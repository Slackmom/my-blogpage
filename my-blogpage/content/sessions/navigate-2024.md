## What This Session Was About

Azure DevOps is powerful. It's also, for many teams, exactly the kind of tool people learn just enough of to survive and never explore beyond that. This session asked: what if we approached DevOps learning like a game?

Not a gamified points system bolted onto a training portal — but actual game design logic applied to how teams discover, practice, and internalize DevOps principles. We looked at what games do well (progressive challenge, meaningful feedback, intrinsic motivation) and mapped it directly to Azure DevOps adoption journeys.

## Key Topics

- Why traditional Azure DevOps training fails to stick
- Game mechanics that map to DevOps learning: leveling up, mastery loops, team quests
- How to design a DevOps learning path that people actually want to follow
- Live demo: a playful approach to Azure Boards and Pipeline exploration
